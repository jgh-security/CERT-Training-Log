# Incident 04 - DVWA File Upload Attack Analysis

## 1. 사건 개요

본 분석은 DVWA(Damn Vulnerable Web Application) 환경에서 제공하는 File Upload 기능을 이용하여 파일 업로드 취약점을 재현하고 웹 서버 로그를 기반으로 업로드 행위를 분석하기 위한 실습 문서이다.

공격자는 파일 업로드 기능의 확장자 검증 미흡 또는 저장 경로 노출 취약점을 이용하여 서버에 파일을 업로드할 수 있으며, 업로드된 파일 접근 여부를 통해 취약 여부를 확인할 수 있다.

본 보고서에서는 정상 파일 업로드를 시작으로 업로드 요청 구조를 확인하고 access log를 기반으로 공격 흐름을 분석할 예정이다.

---

## 2. 분석 환경

| 구분              | 환경                        |
| --------------- | ------------------------- |
| Target          | RHEL                      |
| Attacker        | Kali Linux                |
| Web Server      | nginx                     |
| Web Application | DVWA                      |
| 주요 로그           | /var/log/nginx/access.log |

---

## 3. 공격 재현 과정

### 3.1 정상 파일 업로드 테스트

DVWA의 File Upload 페이지(`/vulnerabilities/upload/`)에서 테스트 파일을 업로드하여 정상 동작 여부를 확인할 예정이다.

업로드 대상 파일:

```text id="x1kt3s"
test.txt
```

업로드 후 `/hackable/uploads/` 경로에서 접근 여부를 확인한다.

---

### 3.2 업로드 파일 접근 확인

업로드된 파일이 웹 브라우저에서 직접 접근 가능한지 확인하여 저장 위치 노출 여부를 분석할 예정이다.

예상 접근 경로:

```text id="v8u7je"
http://TARGET-IP/hackable/uploads/test.txt
```

---

### 3.3 업로드 요청 구조 분석

파일 업로드 과정에서 발생하는 HTTP 요청을 access log를 통해 확인하고 업로드 요청 방식(GET / POST)을 분석할 예정이다.

---

## 4. 웹 로그 분석

### 4.1 Access Log 분석

웹 서버의 `/var/log/nginx/access.log` 로그를 분석하여 업로드 요청이 기록되는지 확인할 예정이다.

확인 대상 패턴은 다음과 같다.

* POST /vulnerabilities/upload/
* /hackable/uploads/

로그 분석을 통해 Source IP, 요청 URI, 업로드 요청 반영 여부를 확인할 예정이다.

---

### 4.2 공격 단계 분석

로그 패턴을 기반으로 다음 단계의 업로드 흐름을 분석할 예정이다.

1. 업로드 페이지 접근

2. 파일 업로드 요청 발생

3. 업로드 파일 직접 접근 확인

---

## 5. 타임라인 재구성

실습 후 access log를 기반으로 다음 항목을 정리할 예정이다.

## 6. IOC (Indicators of Compromise)

실습 과정에서 확인된 값을 기반으로 IOC를 정리할 예정이다.

| 구분        | 값                         | 설명              |
| --------- | ------------------------- | --------------- |
| Source IP | 작성 예정                     | 업로드 수행 IP       |
| 공격 대상 URL | /vulnerabilities/upload/  | File Upload 페이지 |
| 저장 경로     | /hackable/uploads/        | 업로드 저장 위치       |
| 업로드 파일    | test.txt                  | 정상 업로드 파일       |
| 요청 방식     | POST                      | 파일 업로드 요청       |
| 로그 위치     | /var/log/nginx/access.log | 웹 서버 접근 로그      |

---

## 7. 대응 및 개선 방안

### 7.1 확장자 검증 강화

허용 확장자 외 파일 업로드를 제한해야 한다.

### 7.2 저장 경로 분리

웹 루트 외부 저장을 통해 직접 접근을 차단해야 한다.

### 7.3 파일명 난수화

업로드 파일명을 변경하여 예측 가능성을 줄여야 한다.

### 7.4 로그 기반 업로드 모니터링

POST 기반 업로드 요청을 지속적으로 모니터링할 필요가 있다.

---

## 8. 결론

본 주차에서는 DVWA 환경에서 File Upload 기능을 이용하여 정상 파일 업로드 구조를 확인하고 웹 서버 로그를 기반으로 업로드 흐름을 분석할 예정이다.

실습 결과를 기반으로 최종 업로드 패턴과 대응 방안을 정리한다.
