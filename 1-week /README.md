 [1주차] RHEL 시스템 보안 로그 기초 분석 가이드
월요일: 시스템 로그의 흐름 파악 (journalctl)
실습 내용: RHEL에서 발생하는 모든 이벤트를 관리하는 journal 로그 익히기.

주요 명령어:

journalctl -xe: 실시간 로그 확인 (터미널 하나에 띄워놓고 작업 추천)

journalctl -u sshd: SSH 서비스 관련 로그만 집중 필터링

확보할 데이터: journalctl 실행 시 나오는 전체적인 시스템 메시지 화면 캡처.

화요일: 침입 시도의 핵심, /var/log/secure (가장 중요!)
실습 내용: 외부 침입자가 접속을 시도할 때 남는 인증 실패 로그 확인.

실습 방법:

칼리 리눅스 터미널에서 ssh root@레드햇IP 입력.

비밀번호를 3~5번 고의로 틀림.

레드햇에서 tail -n 20 /var/log/secure 명령어로 방금 생긴 로그 확인.

확보할 데이터: Failed password for root from [칼리IP] 문구가 포함된 화면 캡처.

수요일: 시스템 전체 기록 및 필터링 (/var/log/messages)
실습 내용: 보안 외에도 시스템 에러나 변화가 찍히는 종합 로그 확인.

주요 명령어:

cat /var/log/messages | grep -i "error": 에러 로그만 골라보기.

cat /var/log/messages | grep -i "usb": USB 삽입 등 하드웨어 변화 확인.

확보할 데이터: grep 명령어를 통해 특정 정보를 필터링해낸 터미널 화면.

목요일: 접속자 추적 명령 및 데이터 백업 (last, lastb)
실습 내용: 파일 로그가 아닌 명령어 기반의 접속 이력 확인.

주요 명령어:

lastb: 로그인 실패 기록만 요약해서 보기 (공격자 추적 시 필수).

last: 최근 성공한 로그인 기록 확인.

확보할 데이터: lastb 실행 결과(칼리 IP가 실패 기록에 남은 화면).

금요일: 데이터 최종 점검 및 백업
할 일: 월~목 동안 모은 캡처본(이미지)과 로그 텍스트를 USB나 구글 드라이브에 저장.
