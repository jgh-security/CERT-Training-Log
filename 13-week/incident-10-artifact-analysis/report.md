# Incident 10 - Artifact-Based Forensic Analysis

## 1. 사건 개요

본 분석은 침해 이후 시스템 내부에 남은 아티팩트(Artifacts)를 기반으로 공격자의 행위를 추적하고, 로그 및 명령어 기록을 활용하여 침해 흐름을 재구성하는 것을 목표로 한다.

기존에는 웹 로그 및 네트워크 로그 중심으로 공격을 분석하였다면, 본 단계에서는 시스템 내부 흔적을 중심으로 포렌식 관점의 분석을 수행할 예정이다.

---

## 2. 분석 목표

- 시스템 내부 아티팩트 기반 침해 행위 분석
- 사용자 및 공격자 명령어 실행 기록 추적
- 로그인 및 권한 상승 이벤트 분석
- 로그 기반 공격 흐름 재구성
- 타임라인 구성 및 행위 분석

---

## 3. 분석 환경

| 항목 | 내용 |
|------|------|
| OS | Linux (RHEL / Ubuntu) |
| 분석 대상 | bash_history, auth.log |
| 분석 도구 | 기본 Linux 명령어 |
| 분석 방식 | 행위 기반 포렌식 분석 |

---

## 4. 분석 대상

### 4.1 명령어 실행 기록

```text
~/.bash_history
```

- 사용자가 실행한 명령어 기록 확인
- 공격자가 수행한 행위 식별

---

### 4.2 인증 로그

```text
/var/log/auth.log (또는 secure)
```

- 로그인 성공/실패 기록 분석
- 권한 상승(sudo) 이벤트 확인

---

## 5. 분석 계획

본 주차에서는 테스트 환경에서 공격 시나리오를 직접 수행하여 로그를 생성한 후, 해당 로그를 기반으로 포렌식 분석을 진행할 예정이다.

예상 분석 흐름은 다음과 같다.

```text
로그인 시도 → 명령어 실행 → 권한 상승 → 파일 생성 → 공격 행위 수행
```

---

## 6. 예상 결과

- 공격자의 로그인 흔적 식별
- 실행된 주요 명령어 확인
- 권한 상승 여부 분석
- 의심 행위 식별
- 공격 흐름 기반 타임라인 구성

---

## 7. 결론 (예정)

본 분석을 통해 시스템 내부 아티팩트를 활용한 침해 행위 추적이 가능함을 확인하고, 로그 기반 분석에서 포렌식 분석 단계로 확장할 수 있을 것으로 예상된다.

또한 공격자의 행위를 시간 순서로 재구성함으로써 실제 침해사고 대응 관점에서 분석 역량을 강화할 수 있을 것으로 기대된다.
