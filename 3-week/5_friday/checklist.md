# 3-week 실습 역량 습득 결과 요약 (공격 패턴 분석 및 탐지 심화)

본 문서는 3주차 실습을 통해 습득한 네트워크 공격 패턴 식별 및 침해 탐지 역량을 요약한 결과입니다.

---

## 1. SSH Brute Force 공격 패턴 식별 역량

- Kali Linux 환경에서 `hydra`를 활용하여 SSH Brute Force 공격을 재현하고, 다수의 인증 시도가 네트워크 및 시스템 로그에 어떻게 기록되는지 확인하였습니다.
- 동일 Source IP에서 짧은 시간 내 반복적인 TCP 세션 수립 시도가 발생하는 패턴을 식별하고, 이를 정상 로그인 트래픽과 구분할 수 있는 기준을 확보하였습니다.
- 인증 실패 로그(`Failed password`)가 일정 시간 구간 내 연속적으로 발생하는 현상을 분석하여 비정상 접근 시도를 논리적으로 판단할 수 있는 역량을 습득하였습니다.

---

## 2. 네트워크-로그 기반 공격 상관 분석 심화 역량

- Wireshark를 활용하여 SSH 서비스 포트(`tcp.port == 22`) 트래픽을 필터링하고, 반복적인 TCP 3-Way Handshake 발생 여부를 시각적으로 검증하였습니다.
- `tcpdump -i any port 22 -nn -v` 명령을 통해 CLI 환경에서 실시간 패킷 유입을 모니터링하며 공격 시점과 시스템 로그 생성 시점을 정밀 대조하였습니다.
- 패킷 흐름 상의 반복 연결 시도와 `/var/log/secure` 내 인증 실패 로그의 시간적 일치성을 확인하여 공격 행위의 인과관계를 객관적으로 입증하였습니다.

---

## 3. 로그 기반 침해 탐지 기준 수립 역량

- `/var/log/secure` 로그 분석을 통해 다음과 같은 탐지 기준을 도출하였습니다.
  - 단일 IP에서 다수의 인증 실패 발생
  - 짧은 시간 내 반복 로그인 시도
  - root 계정 대상 집중 공격 여부
- `lastb` 명령을 활용하여 실패 로그인 기록이 `btmp` 데이터베이스에 정상적으로 저장되었는지 검증하고, 사후 증적 확보 절차를 수행하였습니다.
- 단순 이벤트 확인을 넘어, 반복성·집중성·시간 밀집도(Time Density)를 기반으로 공격 여부를 판단하는 분석 기준을 확립하였습니다.

---

## 4. 대응 관점 확장 역량

- 반복 인증 실패 패턴을 기반으로 자동 차단 정책의 필요성을 인지하고, `fail2ban` 적용 시나리오를 설계하였습니다.
- SSH 포트 변경 및 root 로그인 제한과 같은 하드닝 방안을 검토하여, 탐지 이후 대응 단계까지 사고 흐름을 확장하였습니다.
- 공격 재현 → 탐지 → 대응 방안 도출까지의 전 과정을 하나의 침해 대응 사이클로 구성할 수 있는 역량을 확보하였습니다.

---

## 5. 종합 의견

3주차 실습을 통해 단순 패킷 분석 단계를 넘어, 네트워크 공격 패턴을 체계적으로 식별하고 이를 로그와 연계하여 침해 여부를 판단하는 탐지 중심 분석 역량을 강화하였습니다. 공격 트래픽의 반복성과 시간 밀집도를 근거로 논리적 판단 기준을 수립함으로써, 실제 CERT 환경에서 요구되는 초동 탐지 및 분석 능력을 한 단계 심화하였습니다.
