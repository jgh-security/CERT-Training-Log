# 1주차 실습: RHEL 시스템 보안 로그 분석

본 실습은 침해대응(CERT)의 기초인 시스템 로그 구조를 파악하고, 외부 침입 시 발생하는 흔적을 식별하는 데 목적이 있습니다.

---

### 주간 실습 일정 및 내용

#### **월요일: 시스템 로그의 흐름 파악 (`journalctl`)**
- **실습 목표:** RHEL 통합 로그 관리 도구인 journal 로그 사용법 익히기
- **주요 명령어:**
  - `journalctl -xe`: 실시간 시스템 로그 확인
  - `journalctl -u sshd`: SSH 서비스 관련 로그만 필터링
- **수행 기록:**
  - [ ] 실시간 시스템 메시지 화면 캡처 및 업로드
  - [ ] 특이사항: 

#### **화요일: 인증 보안 로그 분석 (`/var/log/secure`)**
- **실습 목표:** 외부 침입(SSH Brute Force 등) 시 발생하는 인증 실패 흔적 식별
- **실습 방법:**
  1. Kali Linux에서 `ssh root@[RHEL_IP]` 접속 시도
  2. 고의로 비밀번호 3~5회 오입력
  3. RHEL에서 `tail -n 20 /var/log/secure` 명령어로 기록 확인
- **수행 기록:**
  - [ ] `Failed password` 문구 및 Kali IP가 포함된 로그 캡처
  - [ ] 분석 결과: 

#### **수요일: 종합 시스템 로그 필터링 (`/var/log/messages`)**
- **실습 목표:** 시스템 에러 및 하드웨어 변화 로그 탐색
- **주요 명령어:**
  - `cat /var/log/messages | grep -i "error"`: 에러 로그 필터링
  - `cat /var/log/messages | grep -i "usb"`: USB 등 하드웨어 변경 이력 확인
- **수행 기록:**
  - [ ] 특정 키워드 필터링 결과 터미널 화면 캡처
  - [ ] 분석 결과: 

#### **목요일: 접속자 이력 추적 (`last`, `lastb`)**
- **실습 목표:** 명령어 기반의 사용자 접속 기록 및 실패 이력 확인
- **주요 명령어:**
  - `lastb`: 로그인 실패 기록 요약 (공격 IP 식별의 핵심)
  - `last`: 최근 성공한 로그인 세션 확인
- **수행 기록:**
