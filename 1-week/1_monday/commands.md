## 1. 시스템 로그의 흐름 파악 (`journalctl`)
- **실습 목표:** RHEL 통합 로그 관리 도구인 journal 로그 사용법 익히기
- **주요 명령어:** `journalctl -xe`, `journalctl -u sshd`
- **실습 내용:** 시스템의 실시간 메시지를 확인하고 특정 서비스(SSH)의 동작 이력을 추적함.

### 주요 분석 결과
* **커맨드 실행 내역:** `sudo journalctl -xe`를 통해 시스템의 전반적인 활동 로그 확인.
* **권한 상승 기록:** 특정 계정(`jgh`)이 `su` 명령을 통해 `root` 권한으로 전환된 시점과 터미널(`pts/0`) 정보 식별.
* **서비스 상태 모니터링:** SSH 서비스가 `0.0.0.0` 포트 22번에서 정상적으로 리스닝(Listening)을 시작한 기록 확인.

### 수행 기록
- [x] 실시간 시스템 메시지 화면 캡처 및 업로드 완료
- [x] SSH 서비스 필터링 로그 분석 완료

## 2. 외부 접속 및 인증 로그 분석
* **공격자(Kali) 활동:** `192.168.200.80` 주소에서 Target 서버로 SSH 접속 시도.
* **인증 실패 기록:** - 패스워드 오입력으로 인한 `Permission denied` 현상 발생.
    - 로그상에 `Failed password` 문구와 함께 공격자 IP 주소 및 포트 정보가 기록됨.
* **인증 성공 기록:** 최종적으로 정확한 패스워드 입력 시 `Accepted password` 로그가 남으며 세션이 수립됨.

## 3. 최종 요약
1. **로그 정합성 확인:** Kali의 접속 시도 시점과 Target 서버의 로그 기록 시점이 일치함을 확인.
2. **분석 숙달:** `journalctl`의 `-u` 옵션을 활용하여 특정 서비스의 보안 이벤트만 정밀하게 추출하는 기법을 습득함.
