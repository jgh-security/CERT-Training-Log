## 1. 패킷 덤프 분석 (`tcpdump`)
- **실습 목표:** CLI 환경에서 실시간 패킷 캡처 및 저장 기법 익히기
- **주요 명령어:** `sudo tcpdump -i any port 22 -nn -v`
- **실습 내용:** 공격자의 접속 시도 시 발생하는 네트워크 패킷의 흐름을 실시간으로 관찰하고 분석함.

### 주요 분석 결과
* **트래픽 및 플래그 분석:**
    - **[S] (SYN):** 칼리(`192.168.200.80`)로부터 타겟(`192.168.200.81`)으로의 초기 접속 요청 확인.
    - **[P.] (PUSH):** SSH 인증 과정에서 암호화된 데이터(ID/PW 등)가 실시간으로 교환되는 구간 식별.
    - **[R] (RESET) / [F.] (FIN):** 접속 종료 또는 강제 세션 해제 시점 확인.
* **정보 수집 (Banner Grabbing):** 패킷 내 `SSH-2.0-OpenSSH_10.2p1` 문구를 통해 공격자 환경 정보 파악 완료.
* **상관관계 분석:** - `tcpdump`에 패킷이 들어오는 시점과 `/var/log/secure`에 `Failed password` 로그가 찍히는 시점(`11:57:51`)이 일치함.
    - 네트워크 활동(원인)이 시스템 보안 로그(결과)로 기록됨을 데이터로 증명함.

### 수행 기록
- [x] 특정 포트(22) 패킷 시작/종료 화면 분할 캡처 완료
- [x] 타겟 서버 시스템 로그(`tail -f`) 대조 분석 완료

## 2. 실습 기술 팁
* **TCP 3-Way Handshake:** `[S] -> [S.] -> [.]` 순서의 플래그 변화를 통해 연결 수립 과정을 이해함.
* **실시간 상관분석:** 터미널 분할 기능을 활용하여 패킷(`tcpdump`)과 로그(`tail -f`)를 동시 모니터링하는 기법 습득.
