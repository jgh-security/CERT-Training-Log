## 1주차: 시스템 로그 분석 (월요일)

### 1. 시스템 로그 흐름 파악 (`journalctl`)
- **실습 목표:** RHEL 통합 로그 관리 도구 분석 및 프로세스 구조 이해
- **핵심 분석 내용:**
    * **PID 분석:** `sshd-session` 프로세스 ID(9980 등)를 통해 개별 세션 추적 방법 습득.
    * **권한 관리:** `su` 명령어를 이용한 root 권한 획득 로그 확인.

## 2주차: 패킷 덤프 기초 (화요일)

### 1. 패킷 덤프 분석 (`tcpdump`)
- **실습 목표:** 실시간 패킷 캡처 및 시스템 로그와의 상관관계 입증
- **주요 명령어:** `sudo tcpdump -i any port 22 -nn -v`

### 주요 분석 결과
* **네트워크 플래그(Flags) 분석:**
    - **[S] (SYN):** 공격자의 초기 접속 요청 확인.
    - **[P.] (PUSH):** SSH 인증 데이터 전송 구간 식별.
    - **[R] (RESET):** 인증 실패 후 세션 강제 종료 지점 확인.
* **상관관계 분석:** `tcpdump`의 패킷 캡처 시간(11:57:51)과 `/var/log/secure`의 로그 기록 시간(11:58:00)이 일치함을 확인하여 공격 행위 입증.

### 실습 기술 팁
* **Banner Grabbing:** 패킷 분석을 통해 상대방의 SSH 버전(`OpenSSH_10.2p1`) 정보를 얻을 수 있음.
* **TCP 종료 메커니즘:** 비정상적인 종료 시 발생하는 `RST` 플래그의 역할을 이해함.
